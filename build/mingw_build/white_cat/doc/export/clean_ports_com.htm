Nettoyer les ports COM de windows

Sur une machine souvent utilisée avec différents hardwares, il arrive que l'attribution automatique des ports COM pose souci.

En effet, à détection d'un nouveau hardware de type série :

  1. carte DMX sur chipset FTDII ( Enttec Pro / Open / Vinc's USB-DMX512.Online.net)
  2. Arduino ( 2009 et précédentes)
  3. ou tout autre périphériphérique simulant la liaison série

Windows va attribuer un PORT COM fixe.

La numérotation de ces ports COM commence à 3 ( compris) , le 1 et le 2 étant réservés notamment au modem.

Le problème c'est que bien souvent on dépasse la limite de 10, qui est une vraie limite pour certains logiciels dont WhiteCat, ou alors des limites pour certains hardware ( type Jeenode). Du coup le périphérique n'est plus accessible.

Sous IDE arduino des ports COM vont apparaitre, alors que seule l'arduino est branchée. Ces ports COM étant attribués de manière fixe il peut donc rester des traces.

Il y a plusieurs manières d'opérer.

Echanger l'affectation du périphérique:

C'est cette solution que l'on prendra dans l'urgence:

  1. Aller dans le gestionnaire de périphériques;click droit sur Poste de travail > propriétés > gestionnaire de périphériques
  2. Sélectionner dans la catégories PORTS COM le périphérique >click droit propriétés

  1. Onglet Paramètres du port
  2. Clicker Avancé
  3. Sélectionnez dans la liste un autre port COM non attribué, en dessous de 10

Notez qu'ici dans cette capture, j'ai 3 ports utilisés, 4 et 5 sont attribués à d'autres matériels non branchés sur la machine( carte DMX et UNO ). Ils apparaissent en cours d'utilisation, alors qu'ils ne sont pas branchés: c'est l'attribution fixe de windows.

Nettoyer les ports utilisés

Sélectionnez l'affichage des périphérique cachés ( attribués mais pas branchés):

  1. Clickez chaque périphérique dans la rubrique PORT COM et supprimez le ( partir de 3), voir désinstallez le si vous avez accès à internet pour réinstaller plus tard le driver.
  2. Redémarrez windows
  3. Branchez votre interface, qui se verra attribuer un nouveau port com situé en début de liste ( 3 ) ou en plus petite place libre dans cette liste.

Pour XP, suivre ici la procédure pour que les périphériques cachés apparaissent bien: http://support.microsoft.com/kb/315539
