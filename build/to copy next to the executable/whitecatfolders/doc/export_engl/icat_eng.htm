Fantastick-iCat

A bi-directional remote application for , iPad.

Introduction

Remote control would be useful to most of us in our daily lives: being on tour, working in a theatre, being alone on the top of the ladder, modifying memories in a control room way too far from the stage, or in need of a faders/buttons device adapted to the specific needs of the show that is also lightweight….

Fantastick-iCat *is not* a software build for iOs. It is a module inside White Cat, which is working with Fantastick, a very wise open-source application created by Juha Vehviläinen.

Fantastick is what is called a GUI (graphical user interface), or front-end: the application is not computing data. It transmits limited data, and waits for commands.

Fantastick sends over the network, in UDP:

  • detected touch points (5 for , 11 for iPad ), positions and states
  • gyroscope data

Fantastick waits for the network commands from a client application, which will ask it:

  • delare object definition (images / textes / lines)
  • move those objects, scale them, reduce them, color them

When you first run Fantastick nothing will be displayed on the screen, you need to feed it with commands: White Cat will send commands of what to print where.

Fantastick actually only works on the iPod / iPhone / iPad. The preset portage is ok for iPod/Phone, some additional code needs to be written for the iPad screen size.

For those who have android tablets or phones, they may perhaps contribute to its portage under Android, sources being here Github. I think the community is missing some Android developers for this beautiful application!!!

Pre-requisite

iPod / iPhone / iPad Wifi Connection and internet access for the first install and use on iPod/Phone. Be aware you need to be using an iOS version greater than or equal to iOS 4.3. Installation:

Internet access in wifi for the very first run of Fantastick-iCat :

  • download Fantastick from the Appstore (Search > fantastick)
  • all images necessary for iCat needs to be downloaded in RAM of the iPod/Phone. This command will be sent from White Cat, [from CFG menu].

Touring:

Communication is done in wifi. It is advised:

  • to set an ad-hoc connection between iPod/Phone and White Cat's computer.
  • or to use an autonomous router in RJ45 - > wifi, to create your own secured wifi network.

Ad-hoc mode is really the best solution for best performance.

It is definitely not recommended that you use public wifi access:

  • if there is any download or streaming from anybody on the network, communication between White Cat and fantastick will be interrupted. Some packets will be delayed, and screen refreshment on the iPod/Phone will have latencies.

All those communications, to be in real time, are performed with UDP protocol: packets and their reception are not confirmed. The sending program (client) sends data and does not ask that packets that were delivered be confirmed by the receiver (server).

If a packet is lost or truncated, information may be lost or corrupted. This concerns essentially screen refreshment in Fantastick, being disturbed while in objects creation phase. Sending from FS to White Cat are done continuously (fingers position and gyro), so a lost packet will not really affect the ending result.

In ad-hoc mode, you may loose a small number of packets.

First step

Please verify first that White Cat will start the connection to iCat on open: (CFG Menu > General > onStart / iCAT)

Otherwise you will spend a long time trying to make this Fantastick module work.

Do a Save CFG > then close and re open White Cat.

Setting Parameters

Setting up parameters of the communication between White Cat and Fantastick may be puzzling sometimes. Once you have set the network addresses of the different devices, and struggle a little bit with your network, you will easily learn how to do it quickly.

It is recommended to set up fixed addresses to your network adapters. Otherwise you will regularly have to change destinations IP between iPod and your computer.

The iPod/Phone and computer hosting White Cat should be on the same IP universe: aka 192.168.1.X.

See here for HOW-TO set up network settings :ArtNet multi-software Configuration and here How to create an ad-hoc network

White Cat

  • keep computer hosting White Cat connected to your home wifi network
  • launch White Cat
  • connected network adapter IP's appears in CFG menu / Network on the left. Select the one that identifies your wifi adapter.

Fantastick

  • connect to your home wifi network
  • install Fantastick from AppStore (it is a free app)
  • Ipod Parameters > wifi > set a fixed IP for the iPod
  • Ipod Parameters > Fantastick
  • give the ÏP address of your computer hosting White Cat as a target
  • close Ipod Parameters

White Cat

  • type the fixed IP of your iPod/Phone (this appears in Ipod Parameters > Wifi > Choosen wifi connection > parameters ), then click the IP box to assign it

  • save with save cfg box, quit White Cat
  • launch Fantastick

Junction

  • be sure that the two machines are sending data to the correct IPs, and that UDP ports are corresponding (input/output ports )
  • launch White Cat
  • be sure that your firewall is not blocking White Cat on launch
  • in White Cat, open the iCat Builder window
  • touch iPod/Phone screen and authorize your firewall to accept the incoming stream of data
  • in iCat Builder window, you should see data moving when you move your finger on Fantastick

  • click first preset Box in Fantastick-iCat Builder to refresh the first preset on the iPod/Phone

If something prints to screen, it's okay, communication is established !

General Graphic update

  • once 2-way communication is established, go to CFG MENU > Network and click Reload GFX

White Cat will ask Fantastick to download all images necessary for its GUI.

So on this step, the network must allow access for the iPhone/Pod to the internet. Once done, you will no longer need to update graphics, until a new version of White Cat needs it (which will be indicated in the release notes).

Fantastick-iCat configuration menu in [CFG] NETWORK

  • the IP address to send to (iPod/Phone)
  • Server port: port receiving data (White Cat)
  • Client port: port to send to data (Fantastick)
  • Iterations: number of times you will create an object on preset calling. This enables you to double or triple data on new preset calling to avoid lost packets
  • Reload gfx: loading in iPod cache all graphics necessary for Fantastick-iCat

Using Presets

You have 8 preset pages in White Cat, enabling you to build 8 very different interfaces.

You can choose to use already built presets, or to clear them and build an interface dedicated to your needs and preferences.

Presets are saved in shows, and are dependent on a show. They may be recalled thru SAVE menu (0.8.2.2)

Selecting a preset:

  • Click in White Cat the preset box

or

  • in Fantastick use the preset button page

When you select a preset, this one is refreshed: previous preset is deleted in Fantastcik, and White Cat sends to FS the command to create boxes, buttons, texts, ….

Presets are also callable in midi (use midi aassignments) or by Banger.

Erasing a preset:

  • Type [F4] or click CLEAR
  • Select preset
  • Confirm

Copy a preset on another

  • Position yourself on the preset to copy
  • Select [Copy]
  • Click the destination preset
  • Confirm

Editing presets

Each preset may have a screen orientation horizontally or vertically:

This may be changed by clicking [orientation].

To ease pagination of your presets, and the alignment of buttons and faders, there is a grid parameter, to move them from blocs of pixels (here 1, but it may be 2, 5, 10, 15, …) .

To edit the grid parameter: type the desired value (from 1 to 50 ) and click its box.

A command selected in the families of command (in edit mode or not ) will appear in red on the editing deck.

Objects families

There is 3 types of object families:

  • Sliders
  • Buttons
  • Strings: texts output

Edit a family

Click the family, it will appears in RED, showing you can now edit it.

First line show the selected command and the number of commands of this family in the preset.

  • To add a command, click [+].
  • To delete last command click [-].
  • To navigate inside commands, click the arrows.
  • At the creation of a new command, this one is automatically selected.
  • If you want to duplicate a specific command, either in sliders or buttons, select it, and click [DUB]. You will duplicate all parameters of the command (type, action, values) and thus not waste time in clicking the same parameters from one command to the other.

You can now move it onto the desk configuring the iPod/Phone, second row showing coordinates of the command.

A family has the following components:

  • size of the command (small / medium / big), and for sliders a Vertical/Horizontal way.
  • Type of command (Faders / Cue-List / …)
  • Action (Go / Up / Level / …)
  • if needed a value to edit

Tactile usage

Using buttons is simple: press, release.

Concerning sliders, Slider's title appears in red when a finger manipulates it.

iPod/Phone recognizes up to 5 points. When you use commands such as sliders, once a finger is detected and assigned to a slider, it controls only this slider, until you release the finger. This finger may get out of the space of the slider, it will not touch anything else, being already assigned to a command.

Advice

As communication works 2 ways, between White Cat and Fantastick, it is recommended for the moment to create “light” presets, without too many buttons and sliders, or the bandwidth will be saturated if you have a lot of information refreshed (imagine 16 faders in LFOs running all at the same time).
